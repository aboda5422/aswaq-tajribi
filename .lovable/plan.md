# المرحلة الأولى: واجهة العميل - متجر تطبيق اسواق تجريبي

## الهوية البصرية
- **الألوان**: أخضر طازج (#1a3c2a، #2d5a3d، #5a8a5c، #a0c49d) مع خلفية بيضاء ناصعة
- **الخطوط**: Outfit للعناوين + DM Sans للنصوص (عصرية، متوافقة مع العربي والموبايل)
- **التخطيط**: hero-grid مثل نينجا - بانر عروض علوي + شبكة أقسام ومنتجات
- **الاتجاه**: RTL كامل، تصميم متجاوب (جوال + كمبيوتر)
- **الحواف**: مستديرة (12px) - عصرية ومرنة

## حفظ ذاكرة المشروع
حفظ كل بيانات الشركة والمتطلبات في ذاكرة المشروع عشان ما ننسى شي في المراحل القادمة

## الصفحات المطلوبة

### 1. الصفحة الرئيسية
- شريط علوي: لوقو + بحث + اختيار العنوان + سلة + تسجيل دخول
- سلايدر عروض (بانرات ترويجية)
- أقسام المنتجات بالصور (خضار، فواكه، ألبان، مخبوزات، مشروبات، حلويات...)
- قسم "الأكثر مبيعاً"

### 2. صفحة القسم
- شبكة بطاقات المنتجات مع فلترة وترتيب
- إضافة سريعة للسلة من البطاقة

### 3. صفحة المنتج
- صورة + وصف + سعر + زر إضافة مع كمية + منتجات مشابهة

### 4. التسجيل وتسجيل الدخول
- تسجيل بالجوال أو الإيميل
- تسجيل دخول + نسيت كلمة المرور
- الملف الشخصي (اسم، عنوان، جوال)

### 5. سلة التسوق
- المنتجات المضافة + تعديل الكمية + الحذف + ملخص الطلب

### 6. إتمام الطلب
- اختيار/إضافة عنوان التوصيل + وقت التوصيل + تأكيد

### 7. متابعة الطلبات
- قائمة الطلبات + حالة كل طلب (قيد التجهيز، في الطريق، تم التسليم)

## التقنيات
- **Lovable Cloud** (Supabase) للباك إند: قاعدة بيانات + مصادقة + تخزين صور
- بيانات تجريبية للمنتجات في البداية
- المراحل القادمة (مناديب، إدارة، تطبيقات موبايل) تُبنى لاحقاً خطوة بخطوة

- `categories` (name, image, slug, sort_order, is_active)
- `products` (name, description, price, original_price, image, category_id, unit, is_active, is_featured)
- `branch_inventory` (branch_id, product_id, stock_quantity, is_available)

### 1.3 تحديث جدول الطلبات
